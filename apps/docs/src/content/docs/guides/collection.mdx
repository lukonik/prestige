# Collection

Collection is a cornerstone of a Prestige library. This is where you define the pages you want to render
on your documentation website.

## Definition

You define collections in the Prestige vite plugin config. It expects an array of collections. Each collection will
represent a separate page area with a separate navigation link in the top header, which will navigate to a separate page with its own
sidebar. For example, you can have two collections for your project: one for `docs` and one for `api`.

```ts
    ...
        prestige({
            collections:[{
                id:'docs',
                items:[...]
            },
                {
                    id:'api',
                    items:[...]
                }
            ]
        })
    ...
```

Think of each collection as a sidebar of the page structure. Because a sidebar is a UI element, a collection is more general-purpose, so that's why we call it a collection.

## How it works

A collection needs two main fields: the ID and the items array.

### ID

The id field is the unique key that identifies the root folder that Prestige will look in to generate content pages.
For example, `id: "docs"` will look into the folder `src/content/docs`.

### items

The items field accepts an array where you define the hierarchical structure of the collection. Prestige will generate the sidebar and navigation buttons based on that.
Let's review what can be passed in the items array.

### InternalCollectionItem

InternalCollectionItem represents a single page. It will automatically generate a TanStack router file that will have a
transformed version of the markdown file. It has the following options:

#### label

The label to display in the sidebar.

#### slug

This is the slug which will be displayed in the URL, and Prestige will transform it into the file path to fetch the content.
It shouldn't have starting or ending slashes, and there should exist an `.md` or `.mdx` file on that path. The slug should contain 
the full path relative to `src/content`. For example, if the introduction content is in `src/content/docs/introduction.mdx`,
the slug will be `docs/introduction`.

#### example

```ts
items: [
  {
    label: "Introduction",
    slug: "docs/introduction", // src/content/docs/introduction.mdx
  },
  {
    label: "Showcase",
    slug: "docs/showcase", // src/content/docs/showcase.md
  },
];
```

The file extension will be detected automatically.

### ExternalCollectionItem

ExternalCollectionItem can be used to add a link to the sidebar when the page is outside the content directory.
For example, if you have an `about.tsx` route that you want to add in the sidebar, you will declare it like this:

```ts
    {
        label:"about",
        link:"/about"
    }
```

The Prestige plugin won't process this item in any way, so it has to be a valid TanStack route page and not a markdown file.
The link can also be an external link like `https://react.dev`. In that case, the value will be passed to the HTML `a` element.

#### label

Similar to the InternalCollectionItem label field.

#### link

An internal or external link to use for navigation.

### CollectionGroup

CollectionGroup groups the link items. Think of it as a file-system directory. It can be recursively added to one another an infinite number of times. But for good UI/UX guidelines, it is better
to have three or four levels. A CollectionGroup doesn't have its own path in the URL; it is pathless.

#### label: string

The label to display as the header of the group.

#### collapsed: boolean

Whether the group is collapsed by default.

#### items

It can be either an InternalCollectionLink, an ExternalCollectionLink, or a SidebarGroupItem.

#### example

```ts
prestige({
  collections: [
    {
      id: "docs",
      items: [
        {
          label: "Guides",
          items: [
            {
              label: "Introduction",
              slug: "docs/introduction",
            },
            {
              label: "Installation",
              docs: "docs/installation",
            },
            {
              label: "Examples",
              items: [
                {
                  label: "Example 1",
                  slug: "docs/example-1",
                },
                {
                  label: "Example 2",
                  slug: "docs/example-2",
                },
              ],
            },
          ],
        },
      ],
    },
  ],
});
```

#### Autogenerate

CollectionGroups can also be generated automatically by their folder. For that, you need to set the Autogenerate field. Prestige will recursively iterate over the directory
and generate groups automatically. The labels of the page links will be the file names of the markdown files. You can customize it via the page's frontmatter object.

##### Autodiscovery: object
object to configure autodiscovery options

###### directory: string

The directory to use for generation.

##### example

```ts
prestige({
  label: "Guides",
  autogenerate: { directory: "guides" },
});

// the generated output might look like this
 {
          label: "Guides",
          items: [
            {
              label: "Introduction",
              slug: "docs/introduction",
            },
            {
              label: "Installation",
              docs: "docs/installation",
            },
          ],
}
```
