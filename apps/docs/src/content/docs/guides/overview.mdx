# Overview

This page explains how Prestige works. It overviews the core concepts that Prestige is built upon and will help you understand all the features that Prestige provides.

## Content Folder

The content folder should contain all the markdown files that you want Prestige to transform. It will be created automatically by `@lonik/create-prestige`. The full path of the content folder is
`<root>/src/content`. Anything outside this folder won't be analyzed by Prestige.

## Markdown Files

Markdown files will contain two main parts: the content itself and the frontmatter object.

The content will be rendered as HTML markup by Prestige.

The frontmatter object will be used to customize the metadata for the page. For example, to change the title of the page, you can set it in the frontmatter:

```md
---
title: Overview
---
```

## Markdown Type Support

Prestige can transform the following markdown types:

- MD (regular markdown)
- MDX (markdown with JSX) [link](https://mdxjs.com/)
- GFM (GitHub Flavored Markdown) [link](https://github.github.com/gfm/). It is an extension over markdown. For example, by default, the markdown spec doesn't support tables, but GFM does.

The file extension value should be either `.md` or `.mdx`.

## Collections

Prestige ignores your markdown files in `src/content` unless they are specified in the collections array. A collection has to have its own folder in `content` and be defined in the Vite plugin as well.
For example, to have a docs collection in the app, we will create a `src/content/docs` folder and define it in the Prestige plugin:

```ts
    prestige(
        collections:[{
        id:"docs",
        items:[...]
    }])
```

Now we can add as many content pages as we want. For example, this will generate an introduction page with the url `/docs/introduction`.

```md
    file path: (`src/content/docs/introduction.md`)

    # Introduction Page
    I am introduction page
```

```ts
prestige({
  collections: [
    {
      id: "docs",
      items: [
        {
          label: "Introduction",
          slug: "docs/introduction",
        },
      ],
    },
  ],
});
```

Now, let's see how Prestige generates routes based on content files.

## `(prestige)` Folder

TanStack Router's preferred way to create routes is a file-based approach. We follow the same pattern. Prestige creates one route per content file.
The output folder is `routes/(prestige)`. There are three different files that are created based on the route type.

### Content Type

When Prestige generates a route for content, two separate files are generated: one regular file that will set critical options for a route, and one lazy-loaded route that will load the HTML markup.
For example, `introduction.mdx` will have two respective files:

- `docs.introduction.tsx`
- `docs.introduction.lazy.tsx`

So, the URL for this page will be `/docs/introduction`.

### Collection Type

Every collection will have its own layout route that will be used to display collection-based UIs like sidebars, navigation buttons, and more. In our example, the docs collection will create a `docs.lazy.tsx` file.

### Routes are statically analyzed

Prestige routes are treated as regular routes for TanStack Router, so you get the full benefits that TanStack Router gives you, like path type definitions, prerendering, and more.

### `(prestige)` directory is readonly

You shouldn't modify the `(prestige)` directory manually in any way. It should be modified only by the Prestige plugin. You shouldn't add any new routes manually, either.
Prestige might decide to clean the directory and recreate everything from scratch, so your route will be lost.

### How to add a route manually in a collection

Outside of the `(prestige)` folder, you can add as many regular routes as you want. If you want to display a page under a collection, for example, under the `/docs` collection, create a route file with the following name pattern:

`(prestige).docs.[name].tsx`

and add it to the collections array:

```ts
    prestige(collections:[{
        id:"docs",
        items:[{
            label:"custom",
            link:"/docs/[name]"
        }]
    }])
```

:::note
We might change the structure of the `(prestige)` folder in the future. So, in general, ignore the `(prestige)` folder. Think of it like the `routeTree.gen.ts` file.

The `@lonik/create-prestige` script automatically adds readonly settings in VS Code settings the same way as it is for `routeTree.gen.ts`.
:::

## Prestige Vite Plugin

The Prestige Vite plugin does all the heavy lifting: it generates HTML markup from content, builds sidebars, generates TanStack route files, and more. It should be added to the Vite plugins array and can be customized by the config parameter.

## Prestige Shell

Prestige Shell is a React component that displays core UI elements like the header, search, footer, and more. It is defined in the `_root.tsx` file and can be customized via props. For the full list of props, see the [Prestige Shell Reference](/docs/reference/prestige-shell-reference).
