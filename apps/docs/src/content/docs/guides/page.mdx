# Page

A Page is a markdown file that is rendered as HTML markup by the Prestige plugin. It consists of two main parts: the content, where you define text, and the frontmatter data for page metadata customization.

## How to define a page

A page should be defined inside the `src/content` folder. Each page should be attached to a specific collection to be processed. For example, if you want to have an introduction page inside the docs collection, you will add it in the `src/content/docs` folder. A page file can also be deeply nested inside the `docs` folder.

When you add a content page in the content folder, Prestige doesn't process it automatically. The page content must be defined in the collections array as well. After that, Prestige will attach the correct URL to that content page and process it properly.

For example, if we want to display the `introduction.md` file on the `/docs/introduction` URL, we should do two things:

- Add the `introduction.md` file at the following path: `src/content/docs/introduction.md`
- Add an introduction item inside the collection items array

```ts
prestige({
  collections: [
    {
      id: "docs",
      items: [
        {
          label: "Introduction",
          slug: "docs/introduction",
        },
      ],
    },
  ],
});
```

## Format of a content page

Prestige transforms content pages into HTML markup. Under the hood, it uses [unifiedjs](https://unifiedjs.com/) with [mdx](https://mdxjs.com/docs/) to perform the transformation performantly. Currently, the following markdown types are supported:

- MD (regular markdown)
- MDX (markdown with JSX)
- GFM (GitHub Flavored Markdown)

For you, the transformation is seamless, as long as the proper file extension is provided. If you set the `.mdx` file extension, all three will be compiled properly. For the `.md` file extension, GFM and MD are supported.

## Frontmatter data

Besides transforming the content page, you can define a frontmatter object that will be used by Prestige to customize the metadata of a content page. Use YAML syntax to define the frontmatter data:

```md
---
title: Overview
---

# Overview

This is an overview.
```
In this case, the title of the browser tab will be "Overview".

For full frontmatter options, check the [Frontmatter Reference](/docs/reference/frontmatter-reference).


## Table of contents

Prestige will automatically generate a table of contents based on the content page. It uses headings (h1, h2...h6) to generate the table of contents recursively.

## Code highlighting

Prestige uses [prismjs](https://prismjs.com/) to highlight code blocks.

## Add unifiedjs plugins

You can also add `rehype` and `remark` plugins to further transform the content. To do this, add the plugins in the Prestige plugin's `markdown` config:

```ts
import remarkMath from "remark-math";
import rehypeKatex from "rehype-katex";

prestige({
  markdown: {
    remarkPlugins: [remarkMath],
    rehypePlugins: [rehypeKatex],
  },
});
```
