# Margaret Hamilton and the Apollo 11 Mission

## The Birth of "Software Engineering"

When humanity first traveled to the moon, it wasn't just rocket fuel that got us there; it was code. At the center of this monumental achievement was **Margaret Hamilton**, Director of the Software Engineering Division at the MIT Instrumentation Laboratory.

### Key Contributions to the Apollo Mission

Hamilton's team was responsible for the onboard flight software. Their rigorous approach included several groundbreaking concepts:

- **Asynchronous Executive:** The system could interrupt itself to prioritize important tasks.
- **Priority Displays:** Warning the astronauts in real-time.
- **Human-in-the-loop:** Designing systems that assumed humans might make errors.

> "There was no second chance. We all knew that."
> — Margaret Hamilton, on the stakes of Apollo 11.

---

### The 1202 Program Alarm

Minutes before the Eagle lander touched down, the computer threw a **1202** and **1201** alarm.

1. The radar system was feeding the computer too much irrelevant data.
2. The computer became overloaded.
3. The _Asynchronous Executive_ kicked in, dropping low-priority tasks to focus exclusively on the steering and landing math.

Because of this robust error-handling, the landing was a success.[^1]

[^1]: This concept of fault tolerance is a cornerstone of modern software architecture.

### The Code That Saved the Mission (Code Block Example)

The code was written in AGC Assembly language. Below is an actual snippet from the `LUMINARY` source code (the software for the Lunar Module), specifically dealing with the ignition routine.

```bash
# LUMINARY Revision 69 - AGC Source Code
# Routine: IGNITION ALGORITHM

		BANK	33
		SETLOC	P22
		BANK

		EBANK=	IGNAST

IGNITION	TC	INTPRET		# Start interpreter
		VLOAD
			IGNAST		# Load ignition time
		STORE	TIG		# Store as Time of Ignition
		EXIT
		TC	PHASCHNG	# Phase change for restart protection
		OCT	04024

		TC	ENDOFJOB	# Yield to higher priority jobs
```
